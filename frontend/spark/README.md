# Spark UI Prototype (Reference Only)

This directory contains the UI prototype generated by [GitHub Copilot Spark](https://githubnext.com/projects/copilot-spark/) from `docs/spark-ui-brief.md`.

**This code is not runnable standalone.** It depends on `@github/spark`, a GitHub-internal package not available on the public npm registry.

## Purpose

Reference implementation for adapting into `frontend/src/`. Key components to port:

| Spark Component | Target in `frontend/src/` | Notes |
|----------------|--------------------------|-------|
| `MainApp.tsx` | `layout/AppShell.tsx` | Chat view with sidebar, composer |
| `Dashboard.tsx` | `features/dashboard/` | Agent cards, PR cards, stats |
| `ConversationDetails.tsx` | `layout/IntelligencePane.tsx` | Right pane: roster, tasks, batch intelligence |
| `AgentCard.tsx` | `features/agents/` | Agent status + task progress card |
| `PRCard.tsx` | `features/dashboard/` | PR with CI checks, reviewers |
| `StatusBadge.tsx` | `components/` | Shared status indicator |
| `lib/types.ts` | `features/*/types.ts` | Type reference — our domain model types take precedence |

## What needs adapting

1. **`@github/spark/hooks` (`useKV`)** — Replace with our REST API client + React state/context
2. **`window.spark.user()`** — Replace with our user context
3. **Tailwind CSS** — Our project uses custom CSS with CSS variables (see `docs/spark-ui-brief.md` color palette)
4. **Type shapes** — Spark types diverge from our domain model; our `AgentData`/`ConversationData` types win

## What NOT to port

- `hooks/use-agent-activity-simulation.ts` — Demo-only fake data generator
- `components/DemoDataButton.tsx` — Spark KV demo loader
- `components/ui/` — shadcn/ui primitives; evaluate whether to adopt or use our CSS component approach
- `.github/dependabot.yml` — Nested config has no effect; repo-level config is at root
- `vite.config.ts` — Uses Spark-specific Vite plugins

## Integration workflow

Follow the project TDD mandate: design doc -> failing tests -> adapt component -> pass tests.
