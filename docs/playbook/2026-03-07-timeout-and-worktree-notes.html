Timeout And Worktree Notes

Playbook Entry · 2026-03-07

Issue 1: Timeout test mismatch

Symptom: backend pytest failed in timeout kill-path validation.

Cause: timeout exception handling diverged between adapter and mocked coroutine behavior.

Fix: accept both asyncio.TimeoutError and builtin TimeoutError in adapter timeout paths, and align test mock side effect to asyncio.TimeoutError.

Issue 2: Stale worktree shell path

Symptom: make task-shell ... returned getcwd: No such file or directory.

Cause: command launched from a deleted worktree directory.

Recovery: run commands from repo root and regenerate task worktree.

Preventive Rules

  • Always run orchestration make targets from repo root.
  • Keep each worker bound to its generated TASK_PROMPT.md.
  • Treat design/playbook/saga updates as mandatory completion artifacts.
