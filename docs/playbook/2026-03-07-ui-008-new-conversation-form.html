Playbook: New Conversation Form (UI-008)

Playbook Entry · 2026-03-07

TDD Notes

  • Tests written first against the expected UX, not the existing broken behaviour:
    1. "shows new conversation form when + New is clicked" — form appears, no API call yet.
    2. "cancels new conversation form without creating" — Cancel closes form, no API call.
    3. "creates a new conversation when form is submitted" — correct title and workingDirectory passed to createConversation; form dismisses; conversation appears in list.
  • Two existing tests that clicked "+ New" and immediately awaited createConversation were updated to also submit the form via the "Create" button.
  • All 146 tests passed after implementation.

Key Files

  • frontend/src/layout/AppShell.tsx — ConversationCreatorState type, conversationCreator state, openConversationCreator(), submitConversationCreator(), generateDefaultTitle().
  • frontend/src/layout/AppShell.css — .conversation-creator block with same structure as .agent-editor.
  • frontend/src/layout/AppShell.test.tsx — 3 new tests, 2 updated.

Debug Tips

  • If the form does not appear after clicking "+ New", check that onNewConversation in HistoryPane calls openConversationCreator() (not createConversation() directly).
  • If the form closes without creating, ensure submitConversationCreator() is awaited properly (void submitConversationCreator() inside the JSX click handler).
  • Validation errors appear in the global error banner (.app-shell__error) — check errorText state if the banner shows unexpectedly.
  • The creator form pre-fills from localStorage key ao_working_dir. In test environments localStorage is not available, so DEFAULT_WORKING_DIR (/home/user/workspace) is used as fallback.

Testing

  • Run all frontend tests: cd frontend && npx vitest run
  • Run targeted test file: cd frontend && npx vitest run src/layout/AppShell.test.tsx
  • 146 tests, 21 test files, all passing.

Common Issues

  • Form appears but "Create" does nothing: Likely the submitConversationCreator function is checking if (!conversationCreator) return — the state may have been cleared by a re-render. Ensure no unrelated state change triggers a reset.
  • Both form and agent editor open at the same time: They use independent state slices (conversationCreator and agentEditor). This is intentional; close one before opening the other if the UX feels crowded.
