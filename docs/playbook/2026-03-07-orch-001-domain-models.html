Domain Models

Playbook Entry · 2026-03-07 · ORCH-001

What it does

Defines 9 enums and 8 dataclasses that form the canonical data structures for the orchestration core. These models represent agents, conversations, tasks, artifacts, message events, scheduler runs, and resource snapshots.

How to run tests

cd backend && python -m pytest tests/test_orchestrator_models.py -q

43 tests covering enum completeness, dataclass instantiation, optional field defaults, and re-export from __init__.py.

Key design choices

  • Enums inherit from both str and Enum, so their values are plain strings that round-trip directly to/from SQLite TEXT columns.
  • No ORM dependency — pure @dataclass classes with type hints.
  • Fields with SQL DEFAULT values carry matching Python defaults (e.g., priority=100, active=0, depends_on_json="[]").
  • Auto-increment integer PKs (MessageEvent.id, ResourceSnapshot.id) are int | None defaulting to None.
  • Timestamps are str to match SQLite TEXT storage — no datetime parsing at the model layer.

Gotchas

  • Ruff enforces X | Y union syntax over Optional[X] — the auto-fix handles this.
  • Import sorting (I001) requires from __future__ import annotations to be in the right block position. Let ruff fix it.
  • Dataclass fields with defaults must come after fields without defaults. Required fields are ordered first, then defaulted fields.
