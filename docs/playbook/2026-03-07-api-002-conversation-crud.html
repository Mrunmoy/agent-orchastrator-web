Conversation CRUD Endpoints

Playbook Entry · 2026-03-07 · API-002

How to run

nix develop
cd backend
uvicorn agent_orchestrator.api:app --reload

# List conversations
curl http://localhost:8000/conversations

# Create a conversation
curl -X POST http://localhost:8000/conversations/new \
  -H "Content-Type: application/json" \
  -d '{"title": "My Debate", "project_path": "/tmp/project"}'

# Select a conversation
curl -X POST http://localhost:8000/conversations/select \
  -H "Content-Type: application/json" \
  -d '{"conversation_id": "UUID_HERE"}'

# Soft-delete a conversation
curl -X POST http://localhost:8000/conversations/delete \
  -H "Content-Type: application/json" \
  -d '{"conversation_id": "UUID_HERE"}'

# Clear all conversations
curl -X POST http://localhost:8000/conversations/clear-all

How to test

nix develop
cd backend
python -m pytest tests/test_api_conversations.py -q

Key files

  • backend/src/agent_orchestrator/api/routes/conversations.py — All five CRUD endpoints
  • backend/src/agent_orchestrator/api/__init__.py — Router registration
  • backend/tests/test_api_conversations.py — 21 endpoint tests
  • backend/src/agent_orchestrator/storage/db.py — Added check_same_thread parameter

Gotcha: SQLite thread safety

Symptom: sqlite3.ProgrammingError: SQLite objects created in a thread can only be used in that same thread

Cause: FastAPI runs sync endpoint functions in a threadpool worker, but the DatabaseManager connection was created on the main thread.

Fix: Added check_same_thread keyword parameter to DatabaseManager.__init__. The conversations module passes check_same_thread=False when creating its in-memory database. This is safe for single-writer use with WAL mode.

Gotcha: test isolation

Symptom: Tests pass individually but fail when run together because conversations from earlier tests leak.

Fix: An autouse fixture calls _init_db() before each test, which creates a fresh in-memory database. Each test also creates its own TestClient via fixture to get a clean app instance.

Design decisions to revisit

  • Module-level database should be replaced with FastAPI dependency injection for proper lifecycle management.
  • No input validation on title length or project_path format yet.
  • No pagination on GET /conversations.
