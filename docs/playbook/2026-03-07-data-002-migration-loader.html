    conn.execute("INSERT INTO agent ...")
    conn.commit()

print(db.schema_version)  # 1
db.close()

In-memory (for tests)

db = DatabaseManager(":memory:")
db.initialize()

Gotcha: executescript resets foreign keys

Symptom: Foreign key constraints not enforced after initialize().

Cause: SQLite's executescript() implicitly issues a COMMIT and resets PRAGMA foreign_keys to OFF.

Fix: Re-enable PRAGMA foreign_keys = ON after every executescript() call.

Key behaviours

  • Idempotent: initialize() can be called multiple times safely.
  • WAL mode: Automatically enabled on file-backed databases.
  • Foreign keys: Enabled on every connection.
  • Schema version: Tracked via PRAGMA user_version (currently 1).
  • Context manager: Supports with DatabaseManager(...) as db: pattern.
