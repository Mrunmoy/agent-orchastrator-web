Codex Adapter Implementation Notes

Playbook Entry · ADPT-002 · 2026-03-07

Key Difference: Output Structure

Claude CLI returns {"result": "...", "session_id": "..."} at the top level.

Codex CLI returns a nested structure: {"id": "...", "output": [{"type": "message", "content": [{"type": "output_text", "text": "..."}]}]}.

The adapter uses _extract_text() to walk the nested output and concatenate all output_text blocks into a single string.

Key Difference: Session Resume

Claude uses --resume <session_id>. Codex uses --session <session_id>. Both are handled transparently by each adapter's _build_command().

Shared Patterns

  • Both adapters handle asyncio.TimeoutError and builtin TimeoutError in timeout paths.
  • Both kill the subprocess on timeout before re-raising.
  • Both return AdapterResult with consistent status enum values.

Testing Notes

  • 16 tests cover: interface conformance, availability check, prompt building, session resume, timeout, CLI errors, malformed JSON, multi-block text extraction, empty output.
  • All tests mock _run_cli to avoid actual CLI invocation.
