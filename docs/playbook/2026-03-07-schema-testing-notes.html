Playbook Entry · 2026-03-07 · DATA-001

Insight: WAL mode vs in-memory databases

Symptom: PRAGMA journal_mode returns memory on :memory: databases even after applying PRAGMA journal_mode = WAL.

Cause: SQLite in-memory databases always use the memory journal mode; WAL requires a file-backed database.

Resolution: Split WAL tests into two: one checks the pragma is present in the SQL text, the other uses tmp_path for a file-backed database to verify actual WAL activation.

Pattern: Schema test structure

The test suite validates schema at multiple levels:

  • Existence: all 8 tables present after schema application.
  • Structure: column names and order match spec exactly (parametrized).
  • Indexes: all 6 recommended indexes exist (parametrized).
  • Idempotency: schema can be applied twice without error.
  • Constraints: unique event_id enforced, autoincrement works.
  • Integration: cross-table insert/query works (agent + conversation + join).

Recommendation

When adding columns in future migrations, extend the parametrized EXPECTED_COLUMNS dict in test_schema.py so column drift is caught immediately.
