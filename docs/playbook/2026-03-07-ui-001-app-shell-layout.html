Playbook: App Shell Layout (UI-001)

Playbook Entry · 2026-03-07

TDD Notes

  • Tests written first covering: component presence via data-testid, heading text, interactive elements (buttons, inputs), and class name assertions.
  • jsdom does not apply CSS from file imports, so getComputedStyle tests were replaced with className assertions. This is a known limitation — visual layout must be verified via the dev server or screenshot capture.

Key Files

  • frontend/src/layout/AppShell.tsx — root grid container, composes all zones.
  • frontend/src/layout/AppShell.css — shared design tokens and grid definition.
  • frontend/src/layout/index.ts — barrel export for clean imports.
  • Each zone: TopBar, HistoryPane, ChatPane, IntelligencePane, BottomControls with co-located .css files.

Debug Tips

  • If layout breaks at a breakpoint, check AppShell.css media queries (1280px and 930px).
  • The .panel class uses grid-template-rows: auto 1fr — child elements must follow heading + content order.
  • Use make mockup to compare the original static mockup against the React implementation.

Testing

  • Run tests: nix develop --command bash -c "cd frontend && npx vitest run"
  • 36 tests across 7 test files, all passing.
