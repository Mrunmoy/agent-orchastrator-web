Typography

Inter is loaded via @import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap') at the top of AppShell.css. The body font-family falls back through -apple-system, BlinkMacSystemFont, Segoe UI, and system-ui.

Running Tests

cd frontend && npx vitest run

Or via project root:

nix develop --command make test
ThemeSmoke.test.tsx verifies that app-shell, top-bar, history-pane, and run-status testids are present and carry the expected CSS classes. These tests serve as regression guards against accidental removal of structural class names.

Troubleshooting

  • Hardcoded colours appearing: Search for hex literals in CSS files: grep -r '#[0-9a-fA-F]\{6\}' frontend/src --include='*.css'. Replace with appropriate token.
  • Token not resolving: Ensure the component's CSS file is imported in its TSX file, and that AppShell.css (which defines :root tokens) is loaded before any component CSS.
  • Prettier failures: Run cd frontend && npx prettier --write 'src/**/*.css' to auto-fix formatting.
  • Inter font not loading (offline): The Google Fonts import is best-effort; the system-ui fallback stack covers all functional scenarios.
