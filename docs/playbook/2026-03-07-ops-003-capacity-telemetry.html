Capacity Telemetry Snapshot

Playbook Entry · 2026-03-07 · OPS-003

What it does

Reads CPU load and RAM usage via psutil, packages results into a CapacitySnapshot, and evaluates three ordered gates to decide whether a new agent run may be launched.

Gates (checked in order)

  • CPU: 1-min load average as percent of cores ≥ 85% → denied
  • RAM: Free RAM (total − used) ≤ 2048 MB → denied
  • Active runs: Current runs ≥ max allowed → denied

First failing gate produces a CapacityVerdict(allowed=False, reason="..."). All thresholds are configurable.

Testing notes

All 22 tests mock psutil.getloadavg and psutil.virtual_memory. No real system calls in the test suite. Edge cases at exact threshold boundaries are covered.

Known stubs

  • GPU telemetry returns "[]" — real enumeration deferred.
  • agent_capacity_available is caller-supplied; this module does not track running agents.
