The Order of the Forge

Saga Chapter · UI-011 · 2026-03-07

Prologue: The Voice Without Direction

In the Hall of Debate, where agents traded arguments like swords in the lamplight, there sat an old and cumbersome relic: the Target First selector. A hardcoded scroll of three names — All Agents, Codex, Claude — it pointed at agents who might not even be present, and ignored the real roster entirely. The adventurers had long since stopped trusting it.

And beyond the selector, a more fundamental question hung in the air: who speaks first? The orchestrator ran agents in turn order, but the forge had no way to let the user set that order. Agents appeared alphabetically, like books on a shelf nobody had arranged.

The Decision in the Tavern

The decision came quietly, over a rough-hewn table strewn with task cards. Two things needed to happen in this chapter: the old selector had to be cut loose, and a new mechanism had to be forged — one that let the user grip agents by the scruff and drag them into place.

"Cut the dropdown. Give them handles."

It was a clean mandate. The chronicler sharpened the quill.

The Reckoning of the Schema

First, the deep work: the schema needed a new column. The agent table had always known an agent's name, provider, and role — but never its place in line. A new field, sort_order, was carved into the stone of schema.sql, defaulting to zero for all who came before this chapter.

The migration wizard in db.py was given a new spell — _apply_migrations() — that checked each existing vault before adding the column, so old databases would not crumble. New ones would find the column waiting. The spell was idempotent: harmless if cast twice.

The New Decree

A fresh PATCH route was hammered onto the gate of the backend: /agents/{id}/order. Send it a sort order, and the agent's place in the queue would shift. A negative number? The gate held firm and returned a 400. A ghost's name? The gate held firm and returned a 404. Success was rewarded with the full record of the changed agent.

The listing of agents changed too — no longer alphabetical, but ordered by sort_order first, name second. The roster at last reflected the will of the commander.

The Forging of the Badge

On each agent card in the left pane, a small golden disc now gleamed — a tiny circle stamped with a number. 1. 2. 3. One-indexed, as any artisan would count: the first in line wears the crown. The badge sat in the top-left corner of each card, catching the eye before the name.

It was a small detail, the kind that separates a workshop from a cockpit.

The Drag

Then came the drag-to-reorder. No new library was summoned to the project's dependencies — the adventurers already had the ancient HTML5 drag scroll in hand. A drag handle appeared on each card row: the Braille character ⠿, a quiet invitation. Grip it. Drag. Drop.

When an agent was dropped into a new slot, the roster reordered itself instantly — an optimistic move. Behind the scenes, a call to reorderAgent() began racing to the backend to cement the change. If it failed, the console would record the loss, but the view would not flinch.

The state sync was subtle. The roster now kept a local copy of the agents in the order the user had set. When the parent added a new agent, the roster updated without a flicker — using a render-phase ref comparison rather than an effect, keeping the timing clean across test and production alike.

The Silencing of the Selector

The old agent-selector dropdown was removed from the Composer, quietly and without ceremony. The onSend signature shed its second argument — no more targetAgentId trailing behind like an unwanted shadow. A message was simply a message.

The "Target First" dropdown in ChatPane was cut too. The hall grew quieter. The composers were simpler. The agents would be reached through their order in the roster, not through a dropdown nobody trusted.

The Tests That Went Before

In the tradition of the forge, the test-smiths worked first. Eight failing tests were hammered out before a single line of implementation was written: backend tests for the new endpoint, frontend tests for the badge, for the sorted roster, for the drag, and for the absence of the selector.

They all glowed red in the test runner. Then came the implementation. One by one, they turned green. When the full suite of 534 backend tests and 150 frontend tests ran without fault, the chronicler recorded it and moved on.

Epilogue: A Forge in Order

The cockpit grew more deliberate. An engineer could now look at the agent pane and know at a glance who would speak first, and could drag the order into alignment before the next batch began. The schema remembered. The badge announced. The selector was gone.

It was one chapter in a long campaign. The next would come soon enough. For now, the forge was in order.
