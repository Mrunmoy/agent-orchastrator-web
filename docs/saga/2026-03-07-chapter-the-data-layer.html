Chapter: The Data Layer

Saga Chapter · 2026-03-07 · DATA-001

With the foundation triad complete, the project needed memory. Conversations, agents, tasks, and execution runs had to persist across restarts and survive the chaos of multi-agent orchestration.

Eight tables were carved into SQLite, each mapped precisely to the domain model spec. WAL mode was chosen so readers never block the single writer thread that drives batch execution. Every table, column, and index was verified by 30 tests before the schema was declared stable.

A small lesson emerged: in-memory test databases silently ignore WAL mode. The fix was simple—test the pragma text and verify WAL on a real file—but it reinforced a principle: test what you ship, not what your test harness simulates.

The data layer leaves behind:

  • a schema that can be applied idempotently,
  • six query-tuned indexes matching anticipated access patterns,
  • a parametrized test suite that will catch column drift on future migrations.
