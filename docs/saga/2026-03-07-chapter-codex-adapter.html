Chapter: The Second Bridge

Saga Chapter · ADPT-002 · 2026-03-07

With the first adapter bridge to Claude already standing, a second span was raised — this time to OpenAI's Codex CLI.

The pattern held: same base contract, same timeout discipline, same normalized result. But Codex spoke a different dialect — nested outputs, session flags instead of resume flags, response IDs instead of session IDs.

Sixteen tests were written before any production code, and all passed on the first implementation attempt. The adapter contract proved its worth: two providers, one interface, zero orchestrator changes needed.

The project now has:

  • two adapter bridges (Claude and Codex) sharing a common contract,
  • a proven TDD workflow for adapter development,
  • momentum toward the third adapter (Ollama) and the scheduler that will drive them all.
