New Conversation Form (UI-008)

Design Codex Entry · 2026-03-07

Context

The existing "+ New" button in HistoryPane silently created a conversation with an auto-generated title (Conversation N) and the global working directory from localStorage. Users had no opportunity to name the conversation or supply a per-conversation working directory before the API call was made. The issue report noted that clicking "+ New" appeared to do nothing, as no visible feedback occurred when the backend was unreachable and the creation happened to fail silently.

This change replaces the instant-create flow with an inline form that collects both a title and a working directory before creating the conversation, and shifts focus to the newly created conversation on success.

Problem Statement

  • No user-visible feedback when "+ New" is clicked — looks like a broken button.
  • No per-conversation working directory. All conversations shared the single global dir.
  • Auto-generated title gave users no agency over conversation naming.

Solution Design

ElementBehaviour
Click "+ New"Opens inline ConversationCreator form below main area
Title fieldPre-filled with Conversation N; editable; required
Working Directory fieldPre-filled from localStorage; editable; required
"Create" buttonValidates both fields are non-empty, calls API, closes form, selects new conversation
"Cancel" buttonDiscards form, no API call
Validation failureSets errorText in AppShell — rendered in existing error banner

State Model

A new ConversationCreatorState type is held in AppShell:

  • null — form is closed (default)
  • { title: string, workingDirectory: string } — form is open, fields editable

This mirrors the existing AgentEditorState pattern already used in AppShell.tsx. The same CSS pattern (.agent-editor) is replicated as .conversation-creator.

Data Flow

StepActorAction
1UserClicks "+ New" in HistoryPane
2AppShellopenConversationCreator() — sets state with pre-filled defaults
3UserEdits title and/or working directory; clicks "Create"
4AppShellsubmitConversationCreator() — validates fields
5AppShellcreateConversation(title, workingDirectory) — POST /conversations/new
6APIReturns new conversation; POST /conversations/select called
7AppShellAdds conversation to list, selects it, closes form

Decisions

  • Inline form (not modal): Keeps the same interaction pattern as the existing agent editor, which is an inline section appended below the main grid. No new modal infrastructure needed.
  • Required validation instead of fallbacks: Earlier implementation used fallbacks (if empty, use default). This was replaced with explicit validation errors to surface intent — users should always know what title and directory they are creating.
  • generateDefaultTitle(count) helper: Extracted to eliminate title-generation duplication across openConversationCreator, ensureConversation, and future creation paths.
  • Global working directory not removed from TopBar: The global dir continues to seed per-conversation defaults. Future work could deprecate the TopBar field once all conversations carry their own directory.

Follow-Up

  • Consider removing the global working directory from TopBar once per-conversation dirs are established as the primary mechanism.
  • Inline editing of the conversation title after creation (rename).
