App Shell Layout (UI-001)

Design Codex Entry · 2026-03-07

Context

UI-001 implements the foundational Slack-like app shell layout that all subsequent UI features (UI-002 through UI-007) will build upon. The layout defines five structural zones using CSS Grid.

Layout Structure

ZoneComponentGrid PositionPurpose
TopTopBarRow 1Title, working directory, run status, batch controls
LeftHistoryPaneRow 2, Col 1 (340px)Conversation list, agent roster
CenterChatPaneRow 2, Col 2 (1fr)Message stream, composer
RightIntelligencePaneRow 2, Col 3 (360px)Agreement/conflict maps, neutral memo
BottomBottomControlsRow 3Steering notes, preference notes, batch continue

Decisions

  • CSS Grid with grid-template-rows: auto 1fr auto for the outer shell, and a 3-column grid for the main content area.
  • Components use BEM-style CSS class naming (.top-bar__title, .chat-pane__composer).
  • Each component has its own CSS file; shared tokens live in AppShell.css.
  • Responsive breakpoints: 3-column above 1280px, 2-column at 930-1280px, 1-column below 930px.
  • Color palette matches the Slack-like mockup (src/mockup.html): purple sidebar, white chat, light intelligence pane.

Rationale

  • CSS Grid chosen over flexbox for predictable 2D layout with fixed sidebars and fluid center.
  • Separate CSS files per component keep styles co-located and reduce merge conflicts when parallel UI tasks land.
  • Barrel export (index.ts) provides clean import path for downstream features.
