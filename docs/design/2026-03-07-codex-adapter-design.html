Codex CLI Adapter

Design Codex Entry · ADPT-002 · 2026-03-07

Context

The Codex CLI adapter (ADPT-002) brings OpenAI Codex CLI parity alongside the existing Claude adapter. Both adapters share the same BaseAdapter contract and produce normalized AdapterResult objects.

Decisions

  • Codex CLI is invoked via codex --json <prompt>, mirroring the structured output approach used by the Claude adapter.
  • Session resume uses --session <id> flag (Codex CLI convention) rather than Claude's --resume flag.
  • Response parsing extracts text from the nested output[].content[].text structure, concatenating multiple output_text blocks.
  • Session ID is extracted from the top-level id field in the JSON response.

Rationale

  • Maintaining adapter parity ensures the orchestrator scheduler can treat all adapters uniformly without provider-specific branches.
  • Codex's nested output structure requires a dedicated _extract_text helper, keeping parsing logic clean and testable.
  • Same timeout and error handling patterns as Claude adapter reduce cognitive load and ensure consistent resilience behavior.
