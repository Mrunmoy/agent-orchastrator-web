Design Codex: UI Modernisation — Engineering Cockpit Theme (UI-012)

Date: 2026-03-07 · Slice: UI-012 · Status: Implemented

Problem

The original UI used a Slack-derived purple palette (#6C5CE7) mixed with light-background panels and inconsistent hardcoded hex values scattered across 16+ CSS files. The result was a low-contrast, visually noisy interface with no cohesive design language. The engineering cockpit metaphor demands a deep-navy base, violet brand accents, and consistent surface layering to communicate depth and focus.

Goals

  • Replace all hardcoded hex colours with CSS custom properties (tokens).
  • Establish a deep navy / indigo background system (--bg, --surface, --surface-2, --surface-3).
  • Adopt Inter as the primary typeface for a modern developer-tool aesthetic.
  • Add smooth 200 ms transitions on interactive elements.
  • Introduce brand-coloured focus rings using box-shadow for accessibility.
  • All existing tests must continue to pass; add 3–4 new structural smoke tests.

Panel Modernisation

  • AppShell — .app-shell grid container; .panel shared class for all three column panels; font family set to Inter via Google Fonts @import.
  • TopBar — deep-surface card (var(--surface)) replacing the old purple header; status pill with animated dot, brand-blue CTA buttons.
  • HistoryPane — conv-row with left-border brand accent on active selection; conversation list uses --surface-2 background.
  • ChatPane — --bg stream area for maximum contrast against message bubbles; user bubbles in brand-tinted surface; agent bubbles in --surface-2.
  • IntelligencePane — cards with box-shadow for depth; --teal left accent on memo cards, --ok on agreement, --warn on disagreement.
  • BottomControls — grid layout with labelled field columns; textarea focus ring in brand colour.
  • AgentCard — hover border highlight using --brand; gold order badge; gradient avatar backgrounds; provider-coloured badges.
  • Composer — brand-coloured send button; consistent border-radius and transition timing.

Typography

Testing Approach

  • All existing Vitest / Testing Library tests must pass unchanged — only CSS is modified, no TSX structure changes.
  • New smoke tests in ThemeSmoke.test.tsx verify that app-shell, top-bar, history-pane, and run-status testids are present and carry the correct CSS classes.
  • Existing AppShell.test.tsx receives two additional structural assertions about app-shell and top-bar classes.
  • Run make test, make lint, make format-check to confirm green.
