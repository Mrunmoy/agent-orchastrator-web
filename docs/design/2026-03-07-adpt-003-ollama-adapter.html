Ollama Memo Adapter

Design Codex Entry · ADPT-003 · 2026-03-07

Context

The Ollama adapter (ADPT-003) is the third adapter bridge. Unlike the Claude and Codex adapters, which serve as debate participants, the Ollama adapter generates neutral decision memos — summaries and analyses of debate progress from a neutral perspective. It calls the Ollama CLI to produce these memos using local LLM models.

Interface Design

Because Ollama serves a different role (stateless memo generation, not session-based debate), it uses a different pattern from the class-based Claude/Codex adapters:

  • OllamaAdapterConfig dataclass with fields: model (default "llama3.2"), host (default "http://localhost:11434"), timeout_seconds (default 120).
  • AdapterResult reuses the shared AdapterResult from adapters.base, always returning session_id=None (stateless).
  • async def run(prompt, *, config=None) -> AdapterResult — Invokes ollama run <model> via asyncio.create_subprocess_exec, passing the prompt via stdin pipe.
  • async def check_available(*, config=None) -> bool — Runs ollama list to verify Ollama is reachable.

Key Differences from Claude/Codex

  • No session concept — each call is stateless; session_id is always None.
  • Prompt via stdin — prompt is piped to ollama run via stdin, not passed as a CLI argument.
  • Module-level functions — uses run() and check_available() functions with a config dataclass, rather than inheriting BaseAdapter.
  • Plain text output — Ollama returns plain text, not JSON. No parsing required.
  • OLLAMA_HOST environment variable — the host config is passed via the OLLAMA_HOST environment variable to the subprocess.

Rationale

  • Module-level functions suit the stateless memo role — no instance state to track between calls.
  • Piping prompt via stdin avoids shell escaping issues with complex debate transcripts that may contain special characters.
  • Reusing AdapterResult from base.py keeps the result contract uniform across all adapters.
  • The check_available() function enables the orchestrator to gracefully skip memo generation when Ollama is not installed.
