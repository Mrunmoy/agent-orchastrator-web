Foundation And Adapter Slices

Design Codex Entry · 2026-03-07

Context

Three foundational slices reached completion in parallel: SETUP-001 backend package layout, SETUP-002 frontend TypeScript shell, and ADPT-001 Claude CLI adapter baseline.

Decisions

  • Backend layout uses a strict src/ package boundary and module domains: adapters, API, orchestrator, runtime, storage.
  • Frontend baseline uses React + Vite + TypeScript for fast iteration and test harness compatibility.
  • Claude adapter outputs normalized AdapterResult objects and supports session resume semantics.

Rationale

  • src/ layout prevents import ambiguity and scales with module growth.
  • Vite + Vitest gives low-friction loop for UI-first milestones.
  • Common adapter interface keeps orchestrator logic decoupled from provider-specific details.

Follow-Up

  • Implement Codex adapter parity (ADPT-002).
  • Enrich adapter error reporting with stderr context.
  • Build FastAPI skeleton (API-001) on top of the new backend foundation.
