Conversation-Scoped Agents

Design Codex Entry · 2026-03-08 · UI-014

Problem

Agents are currently global. When a user creates two agents in a conversation, both show badge "1" because sort_order defaults to 0. Agents should be per-conversation with a turn_order that determines the round-robin message cycling.

Design

The conversation_agent join table (already in schema) becomes the source of truth for which agents belong to a conversation and their turn order.

API Endpoints

EndpointDescription
GET /conversations/{id}/agentsList agents for conversation, ordered by turn_order
POST /agents (modified)Optional conversation_id; auto-increments turn_order
POST /conversations/{id}/agents/{aid}/removeRemove agent from conversation (not deleted)
PATCH /conversations/{id}/agents/reorderBulk reorder via agent_ids array

Round-Robin Cycling

turn_order=1: calude-code → turn_order=2: codex-code → turn_order=3: review-bot → back to 1
