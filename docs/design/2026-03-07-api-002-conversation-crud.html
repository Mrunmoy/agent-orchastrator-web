Conversation CRUD Endpoints

Design Codex Entry · 2026-03-07 · API-002

Context

The FastAPI app shell (API-001) and SQLite schema (DATA-001) are in place. The frontend needs endpoints to create, list, select, and delete conversations. This is the first feature that wires the API layer to the persistence layer.

Scope

Five endpoints covering full conversation lifecycle management. All operate on the existing conversation table.

Non-Goals

  • No pagination on GET /conversations (few conversations expected initially).
  • No WebSocket push for real-time updates.
  • No hard-delete — all deletes are soft (set deleted_at).

Endpoint Specification

MethodPathBodyResponse
GET/conversations—{"ok": true, "data": {"conversations": [...]}}
POST/conversations/new{"title", "project_path"}Created conversation object
POST/conversations/select{"conversation_id"}Updated conversation object
POST/conversations/delete{"conversation_id"}Deleted conversation object
POST/conversations/clear-all—{"ok": true, "data": {"deleted_count": N}}

Decisions

  • New conversations default to: state="debate", phase="design_debate", gate_status="open", priority=100, active=0.
  • IDs generated via uuid.uuid4(); timestamps via datetime.now(UTC).isoformat().
  • Router lives in api/routes/conversations.py, registered in api/__init__.py.
  • Module-level DatabaseManager(":memory:") for now; will be replaced by proper dependency injection later.
  • GET /conversations excludes soft-deleted rows (WHERE deleted_at IS NULL), ordered by updated_at DESC.
  • POST /conversations/select deactivates all conversations first, then activates the target.
  • 404 errors use error_response() with HTTP status 404.

Test Strategy

Integration tests using fastapi.testclient.TestClient with an in-memory SQLite database. Tests cover: empty list, create, list after create, select, delete, clear-all, 404 cases, and envelope format on every response.

Follow-Up

  • Replace module-level DB with FastAPI dependency injection.
  • Add pagination when conversation counts grow.
  • Add input validation (title length, project_path existence).
