JSONL Event Log Writer/Reader

Design Codex Entry · 2026-03-07 · DATA-003

Context

The domain model spec (docs/specs/12-domain-model.md) defines JSONL companion logs that live alongside the SQLite database. These append-only logs at data/transcripts/<conversation_id>.jsonl and data/audit/scheduler.jsonl serve transcript, debug, and audit export use cases. Each line is a self-contained JSON object with at minimum: event_id, timestamp, event_type, and payload.

Decisions

  • EventLogWriter — append-only writer that opens a file handle, creates parent directories on init, and flushes after every write for atomicity.
  • EventLogReader — stateless reader with read_all(), read_since(event_id), and tail(n) methods. Returns empty list if the file does not exist.
  • Automatic timestamps — append() injects an ISO 8601 UTC timestamp if the event dict does not already contain one.
  • Context manager protocol — EventLogWriter supports with blocks via __enter__/__exit__.
  • Path helpers — conversation_log_path(conversation_id) and scheduler_log_path() return canonical Path objects.
  • Both writer and reader accept str or pathlib.Path for the log path.

Rationale

  • JSONL is human-readable, streamable, and trivially appendable — ideal for append-only event logs.
  • Flush-after-write ensures crash safety at the cost of minor I/O overhead, acceptable for event-rate workloads.
  • Separating writer and reader keeps responsibilities clear and allows independent evolution.
  • read_since() uses a linear scan; this is sufficient for local-first workloads where log files stay small. If performance becomes an issue, an index file can be added later.

Test Strategy

14 tests in test_event_log.py covering: file/directory creation, JSON line validity, automatic and preserved timestamps, multi-append ordering, reader methods (read_all, read_since, tail), empty-list on missing file, context manager lifecycle, path helper correctness, flush atomicity, and Path object acceptance.

Follow-Up

  • Log rotation deferred until file sizes warrant it.
  • Indexed seek for read_since() deferred until performance profiling shows need.
