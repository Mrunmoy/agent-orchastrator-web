Shared Dev Commands

Design Codex Entry · 2026-03-07 · SETUP-003

Context

No unified Makefile targets existed for lint, test, format-check, or running dev servers. Developers had to remember per-directory commands for backend and frontend stacks.

Decisions

  • Nine new targets added to root Makefile: test, test-backend, test-frontend, lint, lint-backend, lint-frontend, format-check, run-backend, run-frontend.
  • Aggregate targets (test, lint, format-check) run both stacks sequentially.
  • All commands assume nix develop shell. Backend uses $(PYTHON) variable; frontend uses npm scripts.

Rationale

  • Makefile already established as the project runner — no need for npm workspaces or shell wrappers.
  • Sub-targets allow running each stack independently for faster iteration.
  • Fail-fast semantics ensure broken checks surface immediately.

Test Strategy

Dry-run verification tests in tests/test_dev_commands.py confirm each make target exists and wires the correct underlying commands. 12 tests total.

Follow-Up

  • CI pipeline integration (deferred).
  • Pre-commit hooks (deferred).
  • make lint and make format-check should be run before merging any branch.
